Purpose

This document contains procedural and functional information for version 2.1 of the WhelanLabs Search Engine Manager. This application was built with the express purpose of providing a simple and inexpensive means for organizations to add a general search capability to their internal web data. The WhelanLabs Search Engine Manager is a windows-based implementation of a combination web crawler and search engine service with a simple administrative user interface. If you are encountering problems that are not covered in the help text, you should try checking with the WhelanLabs Search Engine Manager Forum (located here).

The Administrative GUI will be the manager’s means of running the service. It can be accessed after install by selecting ‘all programs/WhelanLabs/start Search Engine Manager’ from the Windows Start button. Once started, you will see the following UI.

 

Initial Setup

[Click HERE for a flash demo of the initial setup process.]

 

The first task for an administrator to perform after installing the software is to configure the site settings. This task is performed via the administrative console’s ‘Search Engine’ tab. The tasks in this tab fall into two parts, the first part is to set the HTTP port on which the site will run. (The default is 8080, but lots of sites will want to change this to port 80 because that is the default port for a web site.) The second part is to identify the ownership of the site; this information is used in communicating to sites that you choose to index.

 

Figure 1 - The Manager Tab for the Search Engine sub-system

 

 

After the initial configuration is complete, click the ‘(re)Start Search Engine’ button, and then select the ‘launch search engine client’ link to verify that the search engine is running. Selecting the link should produce the following results.

 

 

Figure 2 - Search Engine Main Page

 

In the page above, we can tell that the Search Engine is running on port 8080 of the current host. You can use this URL (with a substitution of ‘localhost’ for your host’s name. For example, you can change “http://localhost:8080” to “http://searchengine.whelanlabs.com:8080” as the link for your users to access the search engine.

 

At this point, queries on the search engine will not produce results. In order to get results, you need to run a crawl.

Running a Crawl

[Click HERE for a flash demo of the initial setup process.]

 

The crawl task goes out to specified sites and indexes the specified content. The purpose of the ‘Crawler’ tab on the Manager is to specify sites to hit (the starting points for obtaining data, the criteria for filtering included and excluded sites, and a traversal depth – a point to stop traversing), and to initiate the action once configured.

 

 

Figure 3 - The Manager Tab for the Crawler sub-system

 

The basic idea to the page is to set the main pages of your organization in the ‘Starting Point’ section, specify the filtering criteria in the ‘Filters’ section. (Usually just make a rule to include all pages in your site, and another to exclude all other pages.) See the Crawler Help Page for more details on generating filters.

 

After the configuration is complete, selecting the ‘Start New Crawl’ button will initiate a crawl. When a crawl is running, the button will be inactive. (You can only run one crawl at a time.) After the crawl is complete the ‘Start New Crawl’ button will become active, and the crawl status will change to ‘New Crawl Available’.

Restarting the Search Engine

Completion of a new crawl is not automatically added to the Search Engine. In order for the Search Engine to be updated with the latest crawl information, the administrator needs to select the ‘(re)Start Search Engine’ button. When that is done, the Search Engine will restart with the new crawl data, and the ‘crawl status’ will change to ‘Crawl is Up-To-Date’.

 

Note: In this release, the administrator needs to remain logged in for the Search Engine and the Crawler tasks to run. Check for new releases for enhancements in this area!

Uninstalling

 If something goes wrong, or you just decide that this software is not for you, there is an uninstaller for this application. The uninstaller can be accessed from the application menu in the start menu, or via Windows ‘Add/Remove Programs’.
