Fields

      Crawl Status: This field identifies the current status of the crawler in relation to the search engine. The supported values for this field are:

¨      Crawl is Up-To-Date: This status indicates that the search engine is using the latest crawl results, and no new crawls are currently running.

¨      Crawl in Progress: This status indicates that new crawl results are currently being generated.

¨      New Crawl Available: This status indicates that a crawl has completed, and that a restart of the search engine will incorporate the new data.

 

      Starting Point URLs for Crawling: This 2-part field identifies the starting points for a crawl. The top portion (the multi-line field) contains starting points. Each starting point that conforms to the filtering criteria, listed below, is loaded by the crawler and indexed; after that, same process is recursively followed for the pages for which the current site has links. This recursion is followed until the recursion depth equals the ‘crawler traversal depth’ mentioned below. The lower part of the field allows edits to be made to the list of starting points. Additional URLs can be added to the list by adding them to the lower field and selecting the ‘Add URL’ button. Additionally, starting points can be removed by selecting the URL(s) in the top field and selecting the ‘Remove URL(s)’ button.

      Crawler Traversal Depth: Specifies the maximum number of traversals that the crawler will make from any given starting point URL. For example, if set to a value of ‘2’, the crawler will process starting point URLs, all pages directly linked to the starting point URL, and all pages linked by those pages, but no further. A value of fewer than 10 is usually sufficient for most organizations.

Buttons

      Start New Crawl: Initiates a new crawl. Is inactive (grey and non-clickable) when a crawl is already underway.

      Stop Crawl: Stops a currently running crawl. Is inactive (grey and non-clickable) when crawls are not running.

      Add URL: Appends a new URL to the list of Starting Point URLs.

      Remove URL(s): Removes the currently selected URL(s) in the top part of the ‘Starting Point URLs for Crawling’ field.

      Set Crawler Credentials: Displays a dialog for adding credentials (ID and Password) for that might require login actions when crawling.

Figure 1 - Crawler Credentials Dialog

 

The crawler credentials dialog takes accepts a row for each site requiring login credentials. The ‘host’ value specifies the name of the host in the crawled URL (if unset, credentials apply to all hosts), the port (if set) limits the logic to services running on the specified port (otherwise, the credentials are used for login challenges from any port on the given host). The ID and password are the credentials that the crawler shall use to access the crawled site.

      Configure Proxy for Crawler: Allows the crawler to operate via an HTTP proxy.

Figure 2 - HTTP Proxy Server Configuration Dialog

 

If enabled (first checkbox), the Hostname and port specify the location of the HTTP Proxy to be used by all crawler communications, and the User Name and Password are the credentials used by the crawler to access the HTTP Proxy Server.

Pull Down Menus

      Inclusion domain: Specifies the host and domain criteria for URL inclusion in crawler results. The value specified in this menu is applied to the starting-point URLs in generating dynamic filter criteria for the crawler. Valid values are:

¨      Include URLs with starting-point hosts: This setting causes the crawler to include pages from any host specified in a starting-point URL. As an example, if a starting-point URL of “http://whelanlabs.home.comcast.net/~whelanlabs/content/TestPage.htm” was entered; all URLs containing “whelanlabs.home.comcast.net” would be included in the search results.

¨      Include URLs with starting-point domains: This setting causes the crawler to include pages from any host specified in a starting-point URL. As an example, if a starting-point URL of “http://whelanlabs.home.comcast.net/~whelanlabs/content/TestPage.htm” was entered; all URLs containing “home.comcast.net” would be included in the search results.

¨      Include URLs with starting-point context roots: This setting causes the crawler to include pages from any host specified in a starting-point URL. As an example, if a starting-point URL of “http://whelanlabs.home.comcast.net/~whelanlabs/content/TestPage.htm” was entered; all URLs containing “whelanlabs.home.comcast.net/~whelanlabs” would be included in the search results.

¨      Include all Hosts: This setting causes the crawler to include pages from any host specified in a starting-point URL. As an example, if a starting-point URL of “http://whelanlabs.home.comcast.net/~whelanlabs/content/TestPage.htm” was entered, a URL of “http://www.google.com/enterprise/gsa/index.html” would be included in the search results even though it had no resemblance to any of the starting-point URLs..

Check Boxes

      Restart search engine when crawl is complete: When selected, will cause the web server running the search engine to be restarted using the new crawler results.

      Skip local files: When selected, will cause the crawler to skip local files. Local files have URLs that start with “file://”.

      Skip dynamic URLs: When selected, will cause the crawler to skip dynamic URLs. Dynamic URLs have URLs that include a ‘question mark’, ‘asterisk’, ‘exclamation point’, ‘at sign’, or ‘equal sign’ character  (?, *, !, @, =).

      Skip FTP sites: When selected, will cause the crawler to skip FTP sites. FTP sites have URLs that start with “ftp://”.

      Skip e-mail links: When selected, will cause the crawler to skip e-mail links. E-mail links have URLs that start with “mailto://”.

      Skip probable loops: When selected, will filter out URLs which contain five or more single slashes. (For example, http://server/1/2/1/2/1 would be filtered out because it would be likely to be part of some type of infinite loop.

Links

      View Crawler Report:  Displays a consolidated report on the most recent (possibly still underway) crawl. The crawler report contains the start and end times for the crawl, the duration and depth of the crawl, the number of pages crawled, the filter criteria used for the crawl, the top results by context root, the number of reported errors, and a listing of the errors encountered.

       View Crawler Log File: Opens the crawler log file in a separate window. (For advanced users.)

       Help for this Page: Opens this page in a separate window.
